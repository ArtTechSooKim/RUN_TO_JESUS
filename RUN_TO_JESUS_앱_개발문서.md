# RUN TO JESUS: The Race Through Time — 앱 개발 문서

> VSCode(Claude Code) 개발 착수용 정리 문서. 새 세션에서 이 문서만 읽어도 프로젝트 맥락을 파악할 수 있도록 작성.

---

## 1. 프로젝트 개요

- **행사명**: RUN TO JESUS: The Race Through Time
- **성격**: 새로운교회 청년 연합수련회 (주니어청년/청년1/청년2 공동체 최초 합동 수련회), 약 300명 규모
- **핵심 메시지**: 히브리서 12:1-2 — 믿음의 경주는 세대를 넘어 이어지며, 우리는 그 바통을 다음 세대에 전하는 증인이다
- **앱 목적**: 참가자들이 공간(스테이션)을 탐험하며 인물 스토리를 만나고, NFC 태깅으로 "믿음의 조각"을 모아 최종 메시지를 완성하는 이머시브 체험 보조 도구
- **개발 목표 수준**: **데모/프로토타입** (7/5 토 17:00 전체 브리핑에서 시연). 300명 실사용 프로덕션 빌드가 아님. 완성도보다 "이런 흐름으로 작동한다"를 보여주는 게 목표

---

## 3. 기술 스택

- **프레임워크**: Expo (React Native)
- **빌드**: EAS 커스텀 개발 빌드 (dev client) — ⚠️ **Expo Go 사용 불가**
  - 이유: NFC 읽기(`react-native-nfc-manager` 등)는 Expo Go가 기본 지원하지 않음
- **NFC**: QR 대신 **NFC 태그**로 최종 확정 (스테이션별 배치)
  - iOS는 NFC 읽기에 제약 있음 — 백그라운드 스캔 불가, 태그 인식마다 명시적 UI가 뜸. 아이폰 참가자 비중 고려해 UX 흐름 설계 필요
- **DB**: MySQL
- **배포/호스팅**: Railway
- **디자인**: Figma (화면 시안 출처)

## 4. 핵심 기능 범위 (MVP)

1. **인트로**: QR코드 스캔 → 웹앱/앱 접속 (10층 본당 상영과 연동)
2. **스테이션 태깅**: NFC 태그 인식 → 해당 스테이션 콘텐츠(인물 스토리, 오디오, 추가 자료, 다음 단서) 노출
3. **믿음의 조각 수집**: 스테이션 완료 시 조각 획득
   - 다니엘 = 믿음
   - 요셉 = 인내
   - 사무엘 = 순종
   - 에스더 = 부르심
   - 바울 = 사명
   - ⚠️ 위 5개 인물 매핑은 초기 기획 문서 기준. **실제 6개 스테이션 구성(방탈출/릴레이/블러핑/믿음의가정/미는챌린지/도미노)과 재매칭 필요** — 아래 5절 참고
4. **수집 현황 확인 화면**: 참가자가 자신이 모은 조각을 확인
5. **전체 완료 시 최종 메시지 노출**
6. **관리자용 실시간 밀집도 대시보드** (스태프 전용, 참가자 앱과 분리)

| 스테이션 | 컨셉/인물 | 담당 | 장소 |
|---|---|---|---|
| 방탈출 | 라합, 노아, 아벨 | 보민 | 다니엘홀 & 사무엘홀 (10층 영관) |
| 릴레이 | 요셉 | 선재 | 요셉홀 (11층 영관) |

## 6. 관리자용 실시간 밀집도 대시보드

### 위치 판단 방식
- GPS는 실내·다층 건물에서 정확도가 낮아 채택하지 않음
- **"마지막 태깅 스테이션 = 현재 위치"**로 간주 (기존 NFC 태깅 로그 재활용, 추가 인프라 최소화)

### 데이터 정의
- **스테이션별 현재 인원** = 최근 N분 이내 해당 스테이션 태깅 후, 다음 스테이션 태깅이 없는 참가자 수 (체류 시간 임계값 N 설정 필요)
- **스테이션별 완료율** = 해당 스테이션 완료 태깅 수 / 전체 참가자 수
- **전체 진행률** = 전체 조각 수집 완료 참가자 수 / 전체 참가자 수

### 화면 구성
- 스테이션별 현재 인원 (숫자/막대)
- 밀집도 색상 표시 (여유/보통/혼잡 3단계)
- 층별 도면 이미지 위 스테이션별 인원 오버레이 (기존 도면 이미지 재활용 가능)
- 전체 진행률 게이지

### 백엔드
- 태깅 로그 테이블에서 "최근 태깅 후 미이동" 인원 집계 쿼리
- 갱신 방식: 폴링(30초~1분 주기) 또는 웹소켓 — 데모 단계에서는 폴링으로 충분
- 참가자 앱과 별도의 관리자 전용 화면/권한 필요

### 시현 단계 범위
- 실데이터 불필요 — 더미 데이터로 스테이션별 인원 임의 세팅
- 라이브 태깅 시연 시 숫자가 실시간으로 바뀌는 모습만 보여줘도 충분
- (스트레치, 우선순위 낮음) 3D 워크스루 위 밀집도 히트맵 오버레이
