## 1. 프로젝트 개요

새로운교회 청년연합수련회(주니어청년/청년1/청년2 공동체 합동) 이머시브 체험 프로그램의 이벤트 동반 앱.
히브리서 12:1-2 중심, "믿음의 경주는 지금도 이어지고 있다"는 메시지를 스테이션 탐험 + NFC/QR 태깅 +
믿음의 조각 수집으로 구현.

- 참가자 규모: ~300명
- 레포: `RUN_TO_JESUS` (GitHub, 포트폴리오 겸용)
- 목표: 7/5(일) 17:00 전체 진장님 브리핑에서 데모 시연 → 컨펌 → 이후 실제 프로덕션 반영
- 현재 단계 스코프: **데모/프로토타입** (프로덕션 완성도 아님)

## 2. 기술 스택

- **프레임워크**: Expo (React Native) — 표준 Expo 워크플로우로 진행 (NFC 전환 전까지 EAS 커스텀 빌드 불필요)
- **태깅 방식**: **QR코드 우선 구현** (7/5 데모용) → **다음 주 NFC로 전환 예정** (NFC 카드 20장 보유, 실물 확보 완료)
  - QR/NFC 전환을 고려해 태깅 이벤트는 `{stationId, payload, userId, timestamp}` 형태로 추상화
  - QR 스캔: `expo-camera` 기반 구현
  - NFC 전환 시점에 `react-native-nfc-manager` + EAS 커스텀 dev client 필요(지금 당장은 아님)
- **백엔드**: MySQL + Railway
- **위치 판단**: GPS 미사용(실내·다층 건물 정확도 낮음) → **마지막 태깅 스테이션 = 현재 위치**로 간주

## 3. 팀 구조

- **SOOKIM (본인)**: 인프라, 빌드 설정(추후 NFC 전환 시 EAS), Railway 배포, DB 스키마 설계
- **비개발자 헬퍼 (Claude 사용)**: 프론트엔드 UI 컴포넌트 (Figma 디자인 + 더미데이터 기반, self-contained 화면 위주)
  - 네이티브 모듈, 인프라는 SOOKIM이 계속 소유

## 4. 스테이션 - 인물 - 담당 - 조각 매핑 (확정, 담당자는 진장님 이름만 — 팀원 배분은 각 진장님 재량)

| 스테이션 | 인물/컨셉 | 담당 진장님 | 장소 | 조각 키워드(임시) | QR ID |
|---|---|---|---|---|---|
| 방탈출-노아방 | 노아 | 보민 | 다니엘홀 (10F 영관) | 순종 | `RTJ:NOAH` |
| 방탈출-극장 | 아벨 | 혜선 | 사무엘홀 (10F 영관) | 믿음의 예배 | `RTJ:ABEL` |
| 방탈출-추가게임 | 라합 | 보람 | 다니엘홀/사무엘홀 (10F 영관) | 용기 | `RTJ:RAHAB` |
| 릴레이 | 요셉 | 선재 | 요셉홀 (11F 영관) | 인내 | `RTJ:JOSEPH` |
| 블러핑 | 야곱 | 은규 | 이삭홀 (10F 영관) | 씨름(돌이킴) | `RTJ:JACOB` |
| 믿음의 가정 | 아브라함 & 사라 | 혜선 | 아가페홀 (10F 영관) | 약속을 붙듦 | `RTJ:ABRAHAM` |
| 미는 챌린지 | 삼손 | 수 | 디모데홀 (10F 영관) | 헌신 | `RTJ:SAMSON` |
| 도미노 | 다윗 | 보람 | 새로운홀 (10F 패션관) | 작은 믿음의 시작 | `RTJ:DAVID` |

+ 인트로/입장 QR 1개 별도 (`RTJ:INTRO`, 11F 본당)

> ⚠️ 인물/조각 키워드는 **임시안**입니다. 토요일 회의에서 방탈출 3개 미니게임(노아방/극장/추가게임) 컨셉이
> 최종 확정되면 업데이트 필요. 혜선/보람 진장님은 각 2개 스테이션 담당(방탈출 내 미니게임 + 본 스테이션).
> 방탈출 미니게임은 팀 수용 제약(한 팀만 입장 가능)으로 여러 개로 늘어난 구조이며, 향후 추가 확장 가능성 있음
> → 스테이션 목록은 하드코딩하지 말고 DB에서 관리 가능한 구조로 설계 권장 (관리자 CRUD는 스트레치, 5번 참고).

- QR코드 9개(인트로 포함) 생성 완료 — `qr_codes/` 폴더, payload 포맷은 `RTJ:{STATION_ID}`
- 전체 인쇄용 시트: `qr_codes/RUN_TO_JESUS_QR_전체.png`

## 6. 관리자용 실시간 밀집도 대시보드

### 위치 판단 방식
- **"마지막 태깅 스테이션 = 현재 위치"**로 간주 (QR/NFC 태깅 로그 재활용, 추가 인프라 최소화)

### 데이터 정의
- **스테이션별 현재 인원** = 최근 N분 이내 해당 스테이션 태깅 후, 다음 스테이션 태깅이 없는 참가자 수
- **스테이션별 완료율** = 해당 스테이션 완료 태깅 수 / 전체 참가자 수
- **전체 진행률** = 전체 조각 수집 완료 참가자 수 / 전체 참가자 수

### 화면 구성
- 스테이션별 현재 인원 (숫자/막대), 밀집도 색상 3단계(여유/보통/혼잡)
- 층별 도면 이미지 위 스테이션별 인원 오버레이 (기존 도면 이미지 재활용)
- 전체 진행률 게이지

### 데모 단계 범위
- 실데이터 불필요 — 더미 데이터로 스테이션별 인원 임의 세팅
- 라이브 태깅 시연 시 숫자가 실시간으로 바뀌는 모습만 보여줘도 충분

## 7. 7/5 브리핑 데모 시나리오 (실제 프로그램 진행처럼 재현)

1. **입장**: QR 찍고 앱 설치 유도 → 인트로 영상 재생 (11층 본당 컨셉)
2. **플레이어 시뮬레이션**: SOOKIM이 직접 참가자처럼 스테이션 2~3곳 QR 스캔 →
   조각 획득 화면 변화를 진장님들 앞에서 실시간으로 시연 (9개 전부 돌 필요 없음)

4. 이후 다른 진장님들에게 컨펌 받는 단계로 진행 예정
