# RUN TO JESUS — 엔딩 진행 가이드

> 전체 진행률 바 & 엔딩 골인 연출 기능의 **실제 구현 기준** 설명서. 원래 기획 스펙은 `RUN_TO_JESUS_진행률바_엔딩연출_스펙.md`에 있고, 이 문서는 그걸 실제로 구현하면서 바뀌거나 추가된 내용까지 반영한 "지금 실제로 이렇게 동작한다"는 최종 기준 문서입니다.

---

## 1. 한 줄 요약

전체 진행률을 러너가 결승선을 향해 달리는 바 형태로 상시 보여주다가, 행사 마지막에 스탭이 버튼 두 개(게임 종료 → 엔딩 시작)를 순서대로 눌러서 본당 화면의 바를 강제로 100%까지 골인시키고 별이 반짝이는 축하 연출을 재생한다.

---

## 2. 화면 구성

| 화면 | 역할 | 접속 방법 |
|---|---|---|
| `/map` (참가자 앱) | 상단에 실시간 진행률 바 표시 (실제 %) | 팀 번호 + 이름으로 일반 로그인 |
| `public/broadcast.html` | 본당 대형 화면용. 평소엔 같은 진행률 바, 엔딩 트리거 시 골인 연출 재생 | 브라우저에서 URL 직접 접속, 또는 앱에서 팀 "100" + 이름 "본당"으로 로그인하면 자동 이동 |
| `/superadmin` | 게임 상태 조작 (진행중/종료/엔딩) | 팀 "100" + 이름 "김수"로 로그인 |

---

## 3. 스탭 사용법 (당일 실제로 누르는 순서)

1. **최고관리자 로그인**: 본인 휴대폰에서 팀 번호 "100", 이름 "김수"로 로그인 → `/superadmin` 진입
2. 본당 대형 화면 기기는 별도로 팀 번호 "100", 이름 "본당"으로 로그인 → 자동으로 `broadcast.html`로 이동, 그대로 계속 띄워둠 (인트로 이후 게임 중간~엔딩까지 이 화면 하나로 쭉 감)
3. 참가자 전원이 본당에 집결하면, 휴대폰에서 **"🔌 게임종료상태 (전체 잠금)"** 버튼을 누름 → 모든 참가자 화면이 잠기고 "본당으로 돌아가세요!" 메시지가 뜸
4. 엔딩 영상 재생 타이밍에 맞춰, 같은 화면에서 **"🏁 엔딩 시작 (본당 골인 연출)"** 버튼을 누름 (게임종료상태일 때만 눌림) → 확인 모달에서 한 번 더 확인
5. 그 순간 `broadcast.html`의 진행률 바가 실제 값과 무관하게 100%까지 애니메이션으로 차오르고, 러너가 깃발에 도달하며 화면 전체에 별이 반짝이는 연출과 "경주가 완성되었습니다!" 문구가 뜸 (새로고침 전까지 그 상태 유지)

> 게임 종료와 엔딩 시작은 **서로 다른 두 버튼**입니다. 종료가 먼저, 엔딩 시작은 스탭이 원하는 타이밍에 별도로 누르는 액션입니다.

---

## 4. 상태 흐름

```
progress → (게임 종료 버튼) → ended → (엔딩 시작 버튼) → ending
```

- 한쪽 방향으로만 진행되고, 서버에서 순서를 강제합니다 (`ended` 상태가 아니면 엔딩 시작 API가 거부됨)
- `ended`와 `ending` 둘 다 참가자 화면 잠금은 동일 — `ending`은 본당 대형 화면(`broadcast.html`)에만 다른 의미로 작동
- **"모든 팀 진행도 리셋"**(최고관리자 화면)을 누르면 이 상태도 자동으로 `progress`로 되돌아감 — 다음 테스트 라운드 준비용

---

## 5. 원래 스펙에서 실제로 바뀐 것

| 스펙 | 실제 구현 |
|---|---|
| 웜홀(동심원이 계속 퍼지는) 효과로 골인 연출 | **별이 사방에서 반짝이는 효과**로 교체 — 동심원이 `infinite`로 계속 반복되니 한 번 트리거된 이후 계속 퍼지기만 해서 어색했음 |
| 본당 화면은 그냥 `broadcast.html`을 직접 접속 | **"100조 본당" 로그인 바코드 추가** — 앱 로그인 화면에서 팀 100 + 이름 "본당"으로 입력하면 자동으로 `broadcast.html`로 이동 (최고관리자 "100조 김수"와 같은 방식) |
| (스펙에 없음) | **로그아웃 잠김 버그 수정** — 게임종료/엔딩 상태에서 로그아웃하면 로그인 화면 자체가 잠겨서 아무도(최고관리자 포함) 다시 못 들어가던 버그 발견, `/login`은 항상 열리도록 수정 |

---

## 6. 기술 요약 (자세한 건 스펙 문서·`PROJECT_SUMMARY.md` 참고)

- `app_state.game_state`: `'progress' | 'ended' | 'ending'` (MySQL ENUM)
- `POST /api/admin/ending/start`: `ended` 상태일 때만 `ending`으로 전환, 그 외엔 400 에러
- `RunnerProgressBar` (RN 컴포넌트, `src/components/runner-progress-bar.tsx`)와 `broadcast.html`의 바는 같은 시각 사양(알약 트랙 + 러너 배지 + 고정 깃발)을 각자 구현 — 코드는 공유 안 함(하나는 RN, 하나는 순수 JS)
- `/map`의 바는 `/api/stats/overall`을 폴링한 실제 퍼센트, `broadcast.html`의 `ending` 모드는 실제 값과 무관하게 100% 고정
